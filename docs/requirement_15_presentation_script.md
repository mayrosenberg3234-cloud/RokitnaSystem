# דרישה 15 — תסריט הצגה קצר למרצה

1. התחברי בשם המשתמש `architect` ובסיסמה `1234`.
2. הציגי שבסרגל הצד מופיע רק “תיעוד החלטות”, בהתאם להרשאת האדריכלית.
3. בחרי פרויקט, למשל “עיצוב דירת גן”.
4. נסי לשמור טקסט ריק. הציגי את הודעת השגיאה הידידותית ואת מצב `DecisionInvalid`.
5. הזיני החלטה מקצועית מלאה ושמרי אותה.
6. הציגי את הודעת ההצלחה עם חותמת הזמן.
7. הציגי את הטבלה “החלטות קודמות בפרויקט” ואת הטבלה “היסטוריית שינויים בפרויקט”.
8. פתחי את Sequence Diagram והצביעי על:
   `save_decision()` → `_validate_decision_details()` → `get_project()` → יצירת `DecisionLog` ו-`ChangeHistory` → `save_decision_with_history()`.
9. הסבירי: “ההחלטה וההיסטוריה נשמרות בטרנזקציה אחת. אם השמירה נכשלת, לא נשמרים נתונים חלקיים.”
10. הציגי את תרשים המצבים: Draft → Invalid או Saving → Recorded / SaveFailed.
