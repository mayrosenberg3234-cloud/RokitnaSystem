# Rokitna Project Management System

מערכת ניהול פרויקטים למשרד אדריכלות ועיצוב פנים. המערכת מנהלת לקוחות,
פרויקטים, החלטות מקצועיות, היסטוריית שינויים, דוחות ניהול, תשלומים, אבני דרך
והתראות — בארכיטקטורה רב-שכבתית נקייה.

## מטרת הפרויקט

לספק למשרד "רוקיטנה" מערכת מבוססת הרשאות לניהול הליבה התפעולית: ניהול לקוחות
דיגיטלי, תיעוד החלטות מקצועיות עם מסלול ביקורת, והפקת דוחות ניהוליים עם ייצוא
ל-CSV.

## ארכיטקטורה

כל פעולה זורמת דרך השכבות הבאות, ללא דילוגים:

```
GUI (views) → Controllers → Business Logic / Services → Repository → SQLite
```

- אין SQL בתוך ה-GUI.
- כל גישה למסד הנתונים מתבצעת אך ורק דרך `DBRepository`.
- כל פעולה רגישה עוברת בדיקת הרשאה (`utils/permissions.py`).

## התקנה והרצה

נדרש Python 3.12+.

### Windows (הדרך הפשוטה)

לחיצה כפולה על הקובץ `start.bat` — הוא מתקין את הרכיבים בפעם הראשונה ופותח את
המערכת בדפדפן אוטומטית. הסבר מפורט וקצר נמצא בקובץ `RUN_INSTRUCTIONS.txt`.

### הרצה ידנית (כל מערכת הפעלה)

```bash
cd rokitna_system
pip install -r requirements.txt
python init_db.py          # יצירת מסד הנתונים ונתוני הדגמה (רץ גם אוטומטית מתוך האפליקציה)
streamlit run app.py
```

המערכת תיפתח בדפדפן. מסד הנתונים נוצר אוטומטית בקובץ `rokitna.db`.

## הדרישות שמומשו

### דרישה 1 — ניהול תיק לקוח דיגיטלי
מנהל המשרד יכול ליצור, לצפות, לעדכן ולבקש מחיקה של לקוחות. המערכת מונעת כפילות
טלפון ואימייל, מציגה חותמות זמן ואת כל הפרויקטים המקושרים ללקוח. מתוך תיק הלקוח
המנהל יכול גם **לקשר פרויקט חדש ללקוח**: המערכת יוצרת רשומת `Project` ושומרת בה
את `clientId` של הלקוח הנבחר. פרויקט חדש נפתח בסטטוס **פעיל**. בהמשך, המנהל יכול
לעדכן את סטטוס הפרויקט לפי ההתקדמות: פעיל → בהמתנה/הושלם, או בהמתנה → פעיל/הושלם.
פרויקט שהושלם הוא מצב סופי ואינו נפתח מחדש דרך התהליך הרגיל. כל עדכון סטטוס נשמר
בבסיס הנתונים יחד עם `updatedAt` ומוצג בתיק הלקוח. פרויקט חדש נפתח תמיד בסטטוס
**פעיל**, בהתאם למחזור חיי הפרויקט. לאחר אישור מחיקה, לקוח שאינו מקושר לאף פרויקט
נמחק; לקוח שמקושר לפרויקט כלשהו — פעיל, מושהה או שהושלם — מועבר לארכיון כדי לשמור
על היסטוריית הפרויקטים ועל שלמות המידע. לקוח בארכיון מוצג לקריאה בלבד ואינו ניתן
לעדכון, למחיקה חוזרת או לקישור פרויקט חדש.

### דרישה 15 — תיעוד החלטות מקצועיות בפרויקט
אדריכלית מזוהה בוחרת פרויקט קיים, מזינה החלטה מקצועית ושומרת אותה. המערכת בודקת
הרשאת אדריכלית, תקינות תוכן ההחלטה וקיום הפרויקט. ההחלטה נוצרת עם `projectId`,
`createdByUserId` וחותמת זמן. באותה SQLite transaction נוצרת גם רשומת
`ChangeHistory` שמכילה `decisionId`, `projectId`, `createdByUserId`, תיאור וחותמת
זמן. לכן לא ייתכן מצב של החלטה שנשמרה ללא היסטוריית שינוי תואמת. המסך מציג לאחר
השמירה גם את ההחלטות הקודמות וגם את היסטוריית השינויים של הפרויקט. מחזור חיי
תהליך ההזנה ממומש באמצעות `DecisionEntrySession`: טיוטה, לא תקין, שמירה,
תועד, כשל שמירה וסגירה.

התרשימים המעודכנים והמיפוי המלא נמצאים בתיקייה:
`docs/requirement_15_diagrams/` ו-`docs/requirement_15_code_diagram_mapping.md`.

### דרישה 20 — הפקת דוח בקרה ניהולי
מנהל המשרד מבקש דוח בקרה ניהולי. המערכת אוספת מידע מהפרויקטים **הפעילים בלבד** —
אבני דרך, דרישות תשלום, תשלומים והתראות — מאחדת אותם ל**דוח ניהולי אחד**, שומרת
את ההפקה בטבלת `Reports` (כולל `criteria`, `status` וחותמת זמן), מציגה את הדוח על
המסך ומאפשרת ייצוא ל-CSV. הזרימה והאובייקטים תואמים ל-Sequence/Class Diagram של
חלק ב' (`ReportDashboardForm` → `ReportController` → `Report` → `DBRepository`).
המיפוי המלא נמצא ב-`docs/requirement_20_code_diagram_mapping.md`.

### שיתוף פעולה רב-תפקידי (השלמת חוויית המשתמש)
מעבר לשלוש דרישות הליבה, המערכת מספקת חוויה מלאה ומחוברת לכל תפקיד. רכיבים אלו
ממשים מחלקות מתרשימי חלק ב' שלא קודדו קודם (`Meeting`, `ClientInquiry`,
`Document`/סרטוט, `FieldNote`, `Approval`/שינוי).

**לקוח (פורטל עצמי):** צפייה בפרויקטים ובסטטוס ההתקדמות; פגישות עתידיות עם
אפשרות **לאשר** פגישה; צפייה ו**הורדת סרטוטים**; **אישור/דחיית שינויים**
שתועדו; שליחת פניות לנמען נבחר (מנהל/אדריכלית); וקבלת תזכורות מהמשרד.

**אדריכלית (סטודיו אדריכלי):** תיעוד **הערות שטח**, **העלאת סרטוטים**, ו**תיעוד
שינויים** שנשלחים לאישור הלקוח; קבלת תזכורות מהמנהל. (תיעוד החלטות = דרישה 15.)

**מנהל משרד:** יצירת לקוח שמפיקה אוטומטית **חשבון כניסה + סיסמה** לייצוא;
שיוך פרויקטים ועדכון סטטוס; קביעת פגישות; **שליחת תזכורות** ללקוח ולאדריכלית;
**מעקב מלא** אחר כל פעולות האדריכלית והלקוחות; דוח ניהולי (כולל זמני ביצוע)
ו**דוח פיננסי** (עלות לכל חלק בפרויקט, שולם/יתרה, ועלויות שינויים).
